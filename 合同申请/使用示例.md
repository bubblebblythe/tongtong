## 使用AI运行（最简单）

只需告诉AI：

```
请运行依娃Yvonne的合同OA申请
```

或者：

```
请运行这个合同：
C:\交接-悦彤\社群\依娃Yvonne\合同\依娃Yvonne&广西信华服务协议附件.docx
```

AI会自动：
1. 找到合同和ROI文件
2. 提取合同信息
3. 更新REAL_VALUES配置
4. 运行脚本
5. 告诉你去待发箱检查

---

## 检查清单

运行完成后，在OA待发箱检查：

**基础信息**：
- [ ] 合同名称正确
- [ ] 渠道名称正确
- [ ] 合同日期正确

**签约主体**：
- [ ] 我方签约主体正确（人民币→广州豌豆，外币→香港魔力耳朵）
- [ ] 对方签约主体正确
- [ ] 银行账号正确

**财务信息**：
- [ ] 币种正确
- [ ] 汇率正确
- [ ] 票据类型正确（人民币→增值税电子普票，外币→invoice）
- [ ] 开票形式正确（人民币对私→先款后票，其他→先票后款）
- [ ] **发票税率正确**（人民币对私→6%，人民币对公→1%，外币→0%）

**附件**：
- [ ] 合同Word已上传
- [ ] ROI Excel已上传

**确认无误后，点击"提交"。**

---

## 故障排查

### 问题：字段显示错误

**检查**：
1. `run_real.mjs` 里的 REAL_VALUES 是否更新
2. 是否有DOM补填硬编码覆盖了正确值

**解决**：
- 更新REAL_VALUES
- 或让AI重新提取合同信息

### 问题：对方主体未填写

**检查**：
1. 人民币对私：是否搜到"广西信华人力资源有限公司"
2. 其他：OA系统中是否存在该供应商

**解决**：
- 检查合同乙方名称拼写
- 或在OA中先创建该供应商

### 问题：税率不对

**检查**：
1. 币种识别是否正确
2. 对方类型（对私/对公）判断是否正确

**解决**：
- 检查合同中币种关键字
- 手动在待发箱中修改税率
