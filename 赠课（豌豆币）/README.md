# 每周赠课自动生成

每周三跑一次，自动从 BI 下载销售明细，筛选思维/美术赠课数据，填入飞书表格。

## 快速开始

### 每周三操作流程

1. **手动准备飞书 sheet**（2 分钟）：
   - 打开飞书表格：https://my.feishu.cn/sheets/GFu3sySCBhr62ntmKQQcNxv1n4f
   - 找一个最近的"思维 *"tab（不带"本周"的），右键 → 复制工作表
   - 重命名为本周日期，如"思维 5.20-5.26（本周）"
   - 同样复制一个"美术 *"tab，重命名为"美术 5.20-5.26（本周）"

2. **运行自动化脚本**（5 分钟）：
   ```
   all.bat
   ```
   脚本会自动：
   - 从 BI 下载销售明细
   - 筛选本周（上周三~本周二）的思维/美术数据
   - 打开飞书，找到你刚创建的两个"（本周）"sheet
   - 清空旧数据，填入新数据

3. **检查结果**：浏览器会保持打开 30 秒，检查数据是否正确填入

## 流程

1. **下载**：从 Smartbi BI 系统下载销售明细 Excel
2. **处理**：筛选 P 列首签时间在「上周三 ~ 本周二」之间，按学科分思维/美术
3. **填飞书**：打开飞书表格，自动填入数据

## 用法

### 一站式（推荐）
```
all.bat
```
今天是周三跑这个就行，自动算上周三到本周二。

如果不是周三跑，或者想指定结束日期：
```
all.bat 2026-05-26
```

### 分步跑
```
run.bat              下载（默认本周）
run.bat 2026-05-26   下载（指定结束日期）

node process.js              处理（默认本周）
node process.js 2026-05-26   处理（指定结束日期）

node fill-feishu.js              填飞书（默认本周）
node fill-feishu.js 2026-05-26   填飞书（指定结束日期）
```

### 登录态
- **BI 登录态**：复用 `../月度账单自动生成/auth.json`，无需单独登录
- **飞书登录态**：第一次跑 `fill-feishu.js` 时手动登录一次，会保存到 `feishu-auth.json`

## 周期日期逻辑

「每周三发放上周三到本周二的赠课」：
- 周二 = 结束日期
- 周二 - 6 天 = 上周三 = 起始日期
- BI 下载用 2020-01-01 ~ 周二（含滚动订单）
- 处理时用 P 列（首签时间）筛选 上周三 ~ 本周二

## 列映射

**思维**（销售明细 → 飞书）：
- C 豌豆ID → D
- O 首签订单号 → J
- I 渠道名 → L

**美术**（销售明细 → 飞书）：
- D 画啦啦ID → B
- I 渠道名 → C
- O-R（订单号、首签时间、首签月份、首签金额）→ D-G

## 注意事项

1. **飞书 sheet 必须手动创建**：脚本只负责填数据，不会自动创建/重命名 sheet
2. **sheet 命名规范**：必须包含"思维"或"美术"开头 + "（本周）"结尾，如"思维 5.20-5.26（本周）"
3. **首次运行**：BI 和飞书都需要手动登录一次，之后会自动复用登录态
4. **数据校验**：脚本运行完会保持浏览器打开 30 秒，请检查数据是否正确

## 故障排查

- **找不到 auth.json**：运行 `login.bat` 手动登录 BI 一次
- **找不到飞书 sheet**：检查是否手动创建了带"（本周）"的 sheet
- **数据没填进去**：查看 `logs/` 目录的截图，看卡在哪一步
